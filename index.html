Samba CTDB_SOCKET

Security Vulnerability Analysis // Bugzilla #15921

01. OVERVIEW

This repository documents a security issue identified in the CTDB (Clustered Trivial Database) component of Samba, related to the handling of the CTDB_SOCKET environment variable outside of test mode.

The issue was responsibly reported, officially acknowledged, and fixed upstream by the Samba Team.

This repository exists as a technical archive and long-term reference. Publicly indexed and searchable.

03. ISSUE SUMMARY

At the time of discovery, the environment variable:

CTDB_SOCKET

Could be used outside of test mode, despite never being intended for production usage.

This introduced a potential security risk regarding socket security:

  • Socket path influenced externally via CTDB_SOCKET
  • Code applied chown(2) and chmod(2) post-bind
  • Exposed a symlink race window on the filesystem
While no exploit was demonstrated, this represented an unnecessary and avoidable attack surface.

04. TECHNICAL DETAILS

The function ux_socket_bind() performed the following sequence on a UNIX domain socket path:

When the socket path is controllable, post-bind permission changes may allow a symbolic link swap during operation.

This pattern is a well-known class of filesystem race condition.

06. RESOLUTION

The issue was resolved by enforcing original design intent:

CTDB_SOCKET respected ONLY when CTDB_TEST_MODE is enabled

Result: The vulnerable code path is eliminated from non-test environments.
